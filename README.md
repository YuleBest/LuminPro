## 模块介绍  
本模块旨在让你的 Android 设备随时可以使用高亮度显示屏幕内容。无论设备当前亮度如何，只要满足设定的前台触发亮度条件，模块便会自动且平滑地提升屏幕亮度至硬件支持的峰值状态，确保你在强光室外也能获得清晰的显示效果。

## V2.1 核心特性一览  

- **零耗电的底层监听 (inotifyd)**  
  彻底告别传统的死循环和轮询检测耗电。LuminPro 全新采用底层的 `inotifyd` 事件驱动架构，仅在屏幕亮度文件发生修改的瞬间响应，最大程度节省你的设备电量。

- **现代化的 KernelSU WebUI 参数配置**  
  不再需要手动修改枯燥的 `CONFIG.prop` 文本文件！打开 KernelSU 管理器的模块设置，你可以在舒适的暗色可视化面板中实时调节各项亮度参数：
  - `前台最大亮度`：触发亮度提升的源阈值。
  - `峰值最大亮度`：亮度触发后提升到的极致目标值。

- **护眼级平滑亮度提升**  
  亮度瞬间跃升不仅刺眼，还十分突兀——我们为你设计了专属的高刷新率 50 步亮度微调顺滑过渡算法，让屏幕提亮效果更显柔和。

- **勿扰休眠时段**  
  在 WebUI 中随时指定休眠时间段（例如 `1900-0600`）。在设定的夜间休息时间内，模块服务将静默休眠，不再打扰你的睡眠。

- **实时掌控模块状态**  
  支持通过前端面板直接获取监控系统的核心参数，包含了当前界面的实时亮度、底层拉起的在运行 PID 服务状态，并可以随时抓取详细的运行日志供开发者分析。

## 模块工作流程  
1. **安装期自定义校准**  
   - 通过音量键互动脚本，主动测定你设备通用的 `前台亮度阈值` 及 `硬件峰值亮度`，同时支持无缝继承旧版设定的配置以跳过多余校准。
2. **事件监听阶段**  
   - 启动完毕后，`inotifyd` 会作为背景挂起精准监控屏幕亮度的物理底层节点文件变动。
3. **防抖与步进调整机制**  
   - 取代了早期版本的参数紊乱，当前在用户手动横滑亮度控制条停止响应后，算法会经过安全延时后计算准确的终点步进结果点亮屏幕。

## 安装方法  
1. 在支持 WebUI 标准的管理器环境框架下安装本模块的 ZIP 包。
2. 在音量键安装向导中进行简单的第一步初始化按键确认校准（如果已有旧版配置按音量加直接跳过即可）。
3. 重启设备生效。
4. 如需调整参数，只需在管理器中进入模块设置页面进行更改，点击保存**立即生效**。

## 注意事项  
- **长亮降频与烧屏警告**：长期处于高压力的激发峰值亮度，将指数级增加设备的耗电与发热水平，可能会导致设备卡顿并不可逆转地增加 OLED 屏幕烧屏老化几率。
- 请确保写入参数数值为你的设备安全范围内的数据，避免超出你硬件屏幕能承受的上限。
- 只有确保你在完全理解高亮度带来的体验和危害下，方可能长期启用。功能出现阻断或者其他 BUG 时，请自行保存 WebUI 日志通过社区回馈解决。
