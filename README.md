# LuminPro — 日用屏幕亮度强化

**中文简体** 丨 [English](https://github.com/YuleBest/LuminPro/blob/main/README_en.md)

让你的 Android 设备随时可以突破前台亮度上限，将屏幕提升至硬件峰值  
*基于 KernelSU WebUI + inotifyd 事件驱动架构*

</div>

## 模块介绍

无论当前屏幕亮度如何，只要触发亮度超过设定阈值，LuminPro 便会自动且平滑地将屏幕提升至硬件支持的峰值亮度，确保你在强光户外也能获得清晰的显示效果。模块采用 `inotifyd` 底层文件事件驱动，零轮询、零空转，配合 KernelSU WebUI 提供完整的可视化配置与状态监控。

## V2.2 核心特性

- **零耗电底层监听 (inotifyd)**  
  彻底告别死循环与轮询。仅在亮度节点文件发生变更的瞬间响应，守护进程 (`daemon.sh`) 负责持续监视 inotifyd 进程稳定性，确保监听链路永不中断。

- **KernelSU WebUI 可视化配置**  
  在管理器的模块设置页面中，通过暗色面板实时调整所有参数，保存后**立即生效**，无需重启：
  - `前台最大亮度` — 触发提升的亮度阈值
  - `峰值最大亮度` — 提升后的目标硬件亮度
  - `调整步数` — 平滑过渡的步进数（默认 50 步）
  - `休眠时段` — 指定不触发提升的时间范围（如 `2200-0700`）
  - `亮度/最大亮度节点路径` — 支持自定义设备 sysfs 路径
  - `inotify 监测事件` — 自定义 inotifyd 触发事件类型

- **平滑亮度过渡**  
  毫秒级 50 步步进算法，避免亮度瞬间跃升带来的视觉不适。步数可在 WebUI 中自由调整。

- **智能跳过机制**  
  以下情况模块将自动跳过本次提升，避免干扰正常使用：
  - 系统自动亮度已启用
  - 当前正在播放 HDR 内容（可配置）
  - 当前前台应用在黑名单中
  - 处于配置的休眠时段内

- **应用黑名单**  
  在 WebUI 的「应用」页面中管理黑名单，支持按包名搜索、Activity 精准匹配。黑名单内的应用处于前台时，模块不执行亮度提升。

- **一键峰值触发 (boost.sh)**  
  通过管理器的「操作」按钮或 `action.sh` 快捷开关，可即时将亮度拉至峰值；再次触发则恢复原始亮度。

- **实时状态监控与日志**  
  WebUI 状态页实时显示当前亮度值、inotifyd 运行 PID 及服务状态。日志页支持按 `INFO` / `WARN` / `ERROR` 等级过滤，并可一键复制或导出到 `/sdcard`。

## 配置项说明

| 配置项 | 说明 | 默认值 |
|---|---|---|
| `ui_max_bri` | 前台触发阈值（UI 亮度值） | `0`（需手动校准）|
| `max_bri` | 峰值目标亮度 | `0`（需手动校准）|
| `steps_num` | 亮度过渡步数 | `50` |
| `sleep_time` | 休眠时段，格式 `HHMM-HHMM` | 空（禁用）|
| `auto_bri_sleep` | 自动亮度开启时跳过 | `1`（启用）|
| `display_hdr_sleep` | HDR 内容时跳过 | `0`（禁用）|
| `log_max_size` | 日志文件大小上限（KB） | `512` |
| `now_bri_file` | 当前亮度节点路径 | 标准 panel0 路径 |
| `max_bri_file` | 最大亮度节点路径 | 标准 panel0 路径 |
| `blacklist_apps` | 黑名单包名/Activity 列表 | `[]` |
| `inotify_events` | inotifyd 监测事件 | `c` |

## 安装方法

1. 在支持 KernelSU WebUI 的管理器中刷入模块 ZIP。
2. 在音量键安装向导中完成亮度校准（若已有旧版配置，按音量上键直接继承跳过）。
3. 重启设备生效。
4. 在管理器模块设置页调整各项参数，点击保存立即生效。

## 注意事项

> ⚠️ **OLED 烧屏与发热警告**  
> 长期维持峰值亮度会显著加剧 OLED 屏幕老化与烧屏风险，并大幅增加设备发热与耗电。请根据使用场景合理配置，切勿超出设备硬件安全范围。

- 确保填写的亮度数值在设备硬件支持的安全范围内。
- 遇到问题时，请通过 WebUI 日志页导出运行日志后反馈至社区。
